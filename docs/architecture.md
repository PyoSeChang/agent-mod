# 아키텍처

## 전체 구조

```
┌──────────────────────────────────────────────────────┐
│  Minecraft Server (Forge 1.20.1)                     │
│                                                      │
│  ┌──────────────────────────────┐   ┌────────────┐   │
│  │  agent-mod                   │   │terminal-mod│   │
│  │                              │   │ (optional)  │   │
│  │  Fake Player                 │   │            │   │
│  │  Action System               │   │  터미널     │   │
│  │  Observation System          │   │  렌더링     │   │
│  │  Mod Compat Layer            │   │            │   │
│  │  Monitor (모니터링 + 개입)  ──│──→│  전용 탭    │   │
│  │  HTTP Bridge                 │   │            │   │
│  └──────────────┬───────────────┘   └────────────┘   │
└─────────────────┼────────────────────────────────────┘
                  │ HTTP (localhost)
┌─────────────────┼────────────────────────────────────┐
│  Agent Runtime (외부 프로세스, Node.js)                │
│                 │                                     │
│  ┌──────────────┴──────────────────────────┐         │
│  │  Claude Agent SDK                        │         │
│  │  - 자연어 → tool 호출 변환                │         │
│  │  - 행동 요청 → agent-mod HTTP API        │         │
│  │  - 사고/행동 로그 → agent-mod로 스트리밍  │         │
│  └──────────────────────────────────────────┘         │
└──────────────────────────────────────────────────────┘
```

## 모듈 구성

| 모듈 | 책임 | 비고 |
|---|---|---|
| **agent-mod** | 행동 실행 + 모니터링 + 모드호환 | 단일 Forge 모드 |
| **terminal-mod** | 터미널 렌더링 | optional dependency |
| **Agent Runtime** | 자연어 이해, 계획 수립 | 별도 Node.js 프로세스 |

## agent-mod 내부 패키지

```
com.pyosechang.agent
├── core/           # Fake Player, Action, Observation
├── compat/         # 모드 호환 (AE2, Create, Mekanism ...)
├── monitor/        # 로그 수신, 상태 표시, 개입 처리
├── command/        # /agent 명령어
└── network/        # HTTP Bridge
```

## 왜 이렇게 나누는가

| 구성 | 역할 | 이유 |
|---|---|---|
| agent-mod | 마크 월드 안에서 행동 실행 | 모드 API 접근은 JVM 안에서만 가능 |
| monitor (내부) | 모니터링 + 개입 UI | terminal-mod optional dep으로 연동 |
| Agent Runtime | 자연어 이해 + 계획 수립 | Claude SDK 호출은 외부에서 처리 |

## 사용 흐름

```
1. 플레이어: /agent 철 64개 캐와          ← 인게임 채팅으로 명령
2. Agent Runtime: 판단 시작               ← Agent SDK가 사고
3. agent-mod: Fake Player가 행동 시작     ← 월드에서 실제 동작
4. monitor: 로그 스트리밍                 ← terminal-mod 탭 or 채팅
   ┌──────────────────────────────────────────────┐
   │ [사고] 주변 스캔... 철광석 12개 발견 (32,15,-128) │
   │ [행동] 이동 시작 → (32, 15, -128)              │
   │ [행동] 철광석 채굴 중... (3/64)                 │
   │ [사고] 곡괭이 내구도 낮음, 교체 필요             │
   │ [행동] 돌 곡괭이 → 철 곡괭이 교체               │
   │ > 다이아 곡괭이로 바꿔                   ← 개입  │
   │ [사고] 플레이어 지시: 다이아 곡괭이 사용         │
   │ [행동] 다이아 곡괭이로 교체, 채굴 재개           │
   └──────────────────────────────────────────────┘
5. 완료: 채팅에 "[Agent] 철 64개 수집 완료"  ← 결과 알림
```
