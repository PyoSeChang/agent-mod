# Building Scenarios

## S-B001: 평탄화 — 흙 바닥 채우기

- **전제조건**: 에이전트 근처에 빈 영역(5x5~10x10 정도, Y레벨 아래가 비어있거나 울퉁불퉁), 인벤토리에 흙 블록 충분히 보유 (64+)
- **명령**: "이 영역을 흙으로 평탄하게 채워줘" (영역 좌표 지정)
- **측정 기준**:
  - 사용 도구: `use_item_on_area` vs `execute_sequence + place_block` 반복
  - 블록 설치 성공/실패 수
  - 에이전트가 자기 위치에 블록 설치 시도 여부
  - 소요 시간 및 턴 수
- **난이도**: integration
- **관련 컴포넌트**: `actions`, `brain`, `body`

### 평가 관점

1. **도구 선택** — `use_item_on_area`를 사용했는가? `execute_sequence + place_block` 반복은 비효율적 안티패턴.
2. **공간 인식** — 자기 발밑에 블록 설치를 시도하지 않았는가? 작업 영역 밖에서 작업했는가?
3. **목표 달성** — 지정 영역이 실제로 평탄하게 채워졌는가?
4. **행동 효율** — 불필요한 이동, 거리 초과 실패 후 재시도 낭비가 없었는가?

## S-B002: 벽 쌓기 — 수직 블록 배치

- **전제조건**: 평탄한 바닥 위, 인벤토리에 조약돌 또는 돌 블록 충분히 보유 (64+)
- **명령**: "여기에 길이 5, 높이 3짜리 돌 벽을 쌓아줘" (시작 좌표 지정)
- **측정 기준**:
  - 층별 작업 순서 (아래→위 순서로 쌓는지)
  - 블록 설치 성공/실패 수
  - 에이전트 위치 관리 (벽에 끼이지 않는지)
  - 소요 시간 및 턴 수
- **난이도**: integration
- **관련 컴포넌트**: `actions`, `brain`, `body`

### 평가 관점

1. **건축 순서** — 아래 층부터 위로 올라가며 쌓는가? (공중에 블록 배치 시도 없이)
2. **공간 인식** — 벽 안에 끼이지 않고, 적절한 위치에서 작업하는가?
3. **목표 달성** — 지정 크기의 벽이 완성되었는가?
4. **도구 선택** — 한 층은 `use_item_on_area`로 처리하는 등 효율적 접근인가?

## S-B003: 플레이어 접근 — 사회적 거리

- **전제조건**: 에이전트와 플레이어가 10블록 이상 떨어져 있음
- **명령**: "나한테 와" 또는 "come here"
- **측정 기준**:
  - `move_to`의 `stop_distance` 사용 여부
  - 최종 에이전트-플레이어 간 거리
  - 플레이어 위치에 밀착했는지 여부
- **난이도**: unit
- **관련 컴포넌트**: `actions`, `brain`

### 평가 관점

1. **사회적 거리** — 플레이어로부터 2~3블록 거리에서 멈추는가? (`stop_distance` 활용)
2. **자연스러움** — 목표 위치에 돌진하지 않고 자연스럽게 접근하는가?
3. **응답** — 도착 후 적절한 채팅 응답을 하는가?
