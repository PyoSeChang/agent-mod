### 유저 관찰

- 공중에서 떠서 떨리는 문제 해결 안 됨
- 블럭에 겹치는 문제 재발 — 에이전트가 자기 위치에 블록 설치
- 작업을 굉장히 비효율적으로 함
- 월드에딧처럼 블록 한 층이 통째로 세워짐 (이전 세션)
- move_to가 목표에 밀착함 — human-like 거리 유지 안 됨

### 로그 분석 (Claude)

**긍정적:**
- v0.8.2 수정 사항 (장비 가시성, 도리도리 완화) 동작 확인
- 3틱 딜레이 적용으로 execute_sequence 시간 0.0s → 3.5~4.5s 개선
- move_to 2회 모두 성공, stuck 없음

**문제점:**
- `I-001` **PlaceBlockAction이 level.setBlock() 직접 호출** — 바닐라 충돌 체크(엔티티 겹침, 인접 블록 요구) 우회. 자기 위치에 블록 설치 → 끼임/떨림 발생. gameMode.useItemOn()으로 교체 필요.
- `I-002` **SDK가 use_item_on_area 대신 execute_sequence + place_block 선택** — brain 문제. 30블록 설치를 한 번에 시도 → 거리 초과 실패 → 범위 줄여 재시도 반복. 도구 설명/프롬프트 개선 필요.
- `I-003` **move_to에 stop_distance 개념 없음** — 플레이어/엔티티 접근 시 밀착. human-like 거리 유지 불가.
- `I-004` **프롬프트에 human-like 행동 철학 부재** — 공간 인식, 적정 거리, 자연스러운 동작 속도 등의 가이드라인 없음. 개별 케이스마다 프롬프트 추가는 비확장적.
