## Run 1 — S-F001: 밭 일구기

**명령**: "나무 캐서 호미 만들고 강가 주변 땅 경작해"

### Benchmark

| 지표 | 값 |
|------|-----|
| 턴 | 18 |
| 비용 | $0.37 |
| 소요시간 | ~2분 20초 |
| 경작 성공률 | 8/8 (100%) |
| craft 성공 | 4/4 (판자, 막대, 작업대, 호미) |

### 도구 활용

| 도구 | 호출 횟수 | 평가 |
|------|-----------|------|
| `get_observation` | 4 | 1-2회 불필요 |
| `mine_block` | 3 | `mine_area` 미사용 |
| `pickup_items` | 1 | 자동흡수 있는데 수동 호출 |
| `craft` | 4 | 배치 가능했지만 개별 호출 (정상 범위) |
| `place_block` | 1 | 정상 |
| `equip` | 1 | 정상 |
| `use_item_on_area` | 1 | 새 도구 활용, 9칸 중 8칸 성공 |
| `remember_location` | 1 | 정상 |

### 유저 관찰

- 나무 채굴: mine_block 하나씩 사용, mine_area 미활용
- 제작: craft 잘 작동함 (실제 재료 소비 + 결과물 생성)
- 경작: use_item_on_area로 한번에 처리 — 효과적
- 전반: 완수했지만 채굴 부분이 비효율적

### 로그 분석 (Claude)

**긍정적:**
- `craft(count=3)` 배치 제작 활용 (원목 3개 → 판자 12개 한번에)
- `use_item_on_area` 새 도구 즉시 활용 — 8칸 경작 1회 호출
- 작업 계획을 명확히 세움 (로그: "판자→막대→작업대→호미" 순서)

**문제점:**
- `mine_block` x3 개별 호출 (I-003) — 3블록이라 mine_area 효용은 낮지만 습관적 패턴
- `pickup_items(radius=10)` 수동 호출 (I-004) — 자동흡수가 있지만 brain이 모름
- `get_observation` 4회 — 채굴 전후 2회는 불필요

## Run 2 — S-F002: 씨앗 심기

**명령**: "상자에서 씨앗 꺼내서 밭에 심어"

### Benchmark

| 지표 | 값 |
|------|-----|
| 턴 | 26 |
| 비용 | $0.64 |
| 소요시간 | ~2분 36초 |
| 심기 성공률 | 8/8 (100%) |

### 도구 활용

| 도구 | 호출 횟수 | 평가 |
|------|-----------|------|
| `get_observation` | 5 | 2-3회 불필요 |
| `open_container` | 1 | 정상 |
| `click_slot` | 1 | 정상 (quick_move로 44개 이동) |
| `close_container` | 1 | 정상 |
| `execute_sequence` | 5 | 적극 활용하지만 실패 반복 |
| `equip` | 4 | 매번 재장착 필요 — 비정상 |
| `use_item_on` | 4 | area 실패 후 개별 폴백 |
| `use_item_on_area` | 2 (시퀀스 내) | 부분 실패 |
| `move_to` | 2 | 정상 |

### 유저 관찰

- 씨앗 심기가 어색하게 완수됨
- execute_sequence를 적극 쓰긴 하는데 실패해서 재시도가 많음
- 매번 equip을 다시 하는게 이상함

### 로그 분석 (Claude)

**긍정적:**
- `execute_sequence` 적극 활용 — [equip→use_item_on_area] 패턴 시도
- 실패 후 전략 변경 (area → 개별 → 시퀀스 내 개별) — 적응력 있음
- 최종 성공: 8칸 모두 심기 완료

**문제점:**
- `I-001` **use_item_on_area 거리/위치 버그**: 9칸 중 일부만 성공, "failures" 발생
  - 원인 추정: `findStandingNear()`가 farmland 위 위치를 잘못 계산하거나, 이미 근처에 있는데도 walking 시도
  - 에이전트 로그: "일부만 심김. 밭이 아닌 곳에도 포함된 듯"
- `I-002` **equip 반복**: use_item_on 호출 후 mainhand에서 씨앗이 사라짐
  - 원인 추정: `useItemOn()` 후 아이템 스택이 소비되면 mainhand가 EMPTY로 변경됨
  - 정상 동작일 수 있음 (씨앗 1개 소비) — 하지만 스택이 44개인데 1개 사용 후 빠지면 버그
  - **추가 조사 필요**: UseItemOnAction에서 mainHandItem 참조 방식 확인
- 에이전트가 같은 좌표에 중복 시도 (5,62,32에 2번 use_item_on)
- `execute_sequence` 내 sync action 실패 원인이 응답에 불명확
