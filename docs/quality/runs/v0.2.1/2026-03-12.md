## Run 1 — S-F001: 소규모 밭 만들기

**명령**: "나무 캐고 나무 호미 만들어서 강가 주변 땅 경작"

### Benchmark

| 지표 | 값 |
|------|-----|
| 턴 | ~18 |
| 소요시간 | ~240s (4분) |
| 액션 수 | 16 |
| 경작 성공률 | 3/4 (75%) |
| 씨앗 심기 성공률 | 3/3 (100%) |

### 도구 활용

| 도구 | 호출 횟수 | 평가 |
|------|-----------|------|
| mine_block | 4 | I-003 여전 — mine_area 미사용 (소규모라 영향 미미) |
| craft | 4 | count 배치 활용 (oak_planks x3 등) |
| equip | 2 | 전체 스택 이동 정상 — I-002 해결 확인 |
| use_item_on_area | 2 | 경작 75%, 씨앗 100% |
| use_item_on | 3 | 씨앗 개별 심기 (use_item_on_area 대신) |
| pickup_items | 1 | 아직 호출 (이전 버전 빌드) |

### 유저 관찰

소규모(4×3) 영역이라 mine_area/use_item_on_area 미사용이 이해됨. equip 재호출 없이 진행된 건 개선.

### 로그 분석 (Claude)

**긍정적:**
- I-002 해결 — equip 2회로 충분 (이전: 5회+)
- craft 배치 적극 사용 (count=2, 3)
- 전체 흐름 완수

**문제점:**
- 호미 경작 시 1/4 PASS 반환 — crafting_table 위치에 경작 시도 (brain이 place_block 위치를 경작 영역에 배치)

## Run 2 — S-F002: 9×9 대규모 농장

**명령**: "9×9 규모로 경작하고 씨앗 심기"

### Benchmark

| 지표 | 값 |
|------|-----|
| 턴 | ~25 |
| 소요시간 | ~600s (10분) |
| 경작 성공률 | 39/81 (48%) |
| 씨앗 심기 성공률 | 38/81 (47%) |

### 도구 활용

| 도구 | 호출 횟수 | 평가 |
|------|-----------|------|
| execute_sequence | 2 | equip+use_item_on_area 체이닝 — 정상 작동! |
| use_item_on_area | 2 (시퀀스 내) | 9×9 영역 자동 처리 |
| mine_block | 4 | 여전히 mine_area 미사용 |

### 유저 관찰

대규모에서는 execute_sequence + use_item_on_area 잘 사용함. 그러나 이동할 때 정면을 보지 않고 땅을 보면서 걸음 (어색). 9×9 영역에 잔디/물이 있어서 경작 실패율 높음 — 에이전트가 이를 인지하고 대처 못함.

### 로그 분석 (Claude)

**긍정적:**
- execute_sequence 활용 확인 — equip → use_item_on_area 한 번에 체이닝
- 대규모 영역에서 use_item_on_area 자동 서펜타인 정상 작동
- pickup_items 호출 0회 (도구 제거 효과)

**문제점:**
- I-005: grass_block에 호미 PASS — 잔디/꽃 등 식물이 위에 있으면 useItemOn이 식물에 흡수됨. 경작 전 식물 제거 필요
- I-006: MoveToAction에서 lookAt이 다음 waypoint 좌표(발밑 높이)를 바라봄 → pitch가 아래를 향해 땅 응시. 이동 목적지 높이를 eye level로 조정 필요
- I-007: 영역에 water 블록 다수 포함 → 경작/심기 모두 실패. use_item_on_area가 블록 종류를 사전 필터링하거나, 결과를 brain에 상세 보고해서 재시도 유도 필요
- no_path/no_standing_position 실패 다수 — 물 위에 서있을 수 없어서 경로 없음
